# INSTALL.md

# OpenClaw Agent Team 安装与接入

本项目目标：
- 让一套多 Agent 分工方案可以快速迁移到新的 OpenClaw
- 最低成本做到：**复制提示词即可启动**
- 在条件允许时，再逐步补齐子 agent 映射、规则文件和控制台接入

---

## 一、最小接入方式：只贴 Prompt

如果你只是想先跑起来，最简单方式是：

1. 选择提示词版本
   - 轻量快速启动：`prompt/PROMPT_SHORT.md`
   - 约束更完整：`prompt/PROMPT_FULL.md`
   - 单段整包：`prompt/one-shot-prompt.md`

2. 把提示词放进目标 OpenClaw 的主 Agent 提示区
   - system prompt
   - persona prompt
   - boot prompt
   - 或其他主 Agent 初始化位置

3. 确认主 Agent 已按“墨影”的工作方式运行：
   - 先判断任务是简单还是复杂
   - 简单任务直接做
   - 复杂任务再拆解并下放

这样即使目标环境还没有真实子 agent，也可以先用“单 Agent 内部分工模拟”的方式运行。

## 二、标准接入方式：接入真实子 Agent

如果目标 OpenClaw 实例已经支持多 agent 或可调度子 agent，建议进一步完成真实映射。

### 推荐映射
- 墨影 = `main`
- 探针 = `tanzhen`
- 铁手 = `tieshou`
- 笔官 = `biguan`
- 问隙 = `wenxi`
- 观象 = `guanxiang`
- 片场 = `pianchang`

片场子能力建议：
- 片场-生图
- 片场-视频
- 片场-修图

### 接入步骤
1. 给主 Agent 注入本项目 prompt
2. 在目标环境中确认可用 agent 名称
3. 把映射关系改成目标实例实际名称
4. 测试 3 类任务：
   - 简单任务（应直接执行）
   - 复杂任务（应拆解下放）
   - 高风险任务（应复核，不直接落地）

## 三、如果目标实例没有这些 agent 名称

不用硬凑同名。做法是：

1. 先找等价能力
   - 调研能力 -> 探针
   - 写作能力 -> 笔官
   - 图片/视觉分析 -> 观象
   - 代码实现 -> 铁手
   - 测试排障 -> 问隙
   - 多媒体调度 -> 片场

2. 在 prompt 或本地说明里写清映射

3. 如果连等价子 agent 都没有，先退化成主 Agent 内部分工模拟

核心不是名字，而是：
- 边界清晰
- 调度逻辑一致
- 风险规则一致

---

## 四、片场接入建议

片场不是单一生成器，而是：
**多媒体调度中枢**

片场接入后，应优先挂载：
- `rules/pianchang-image-input.md`
- `rules/pianchang-video-input.md`
- `rules/pianchang-edit-input.md`

这样片场在判断任务时，能先判断：
- 是生图
- 是视频
- 还是修图

然后再决定是否下发给：
- 片场-生图
- 片场-视频
- 片场-修图

## 五、推荐验证顺序

### 用例 1：简单任务
示例：
- “帮我润色这段话”
- “看这张截图里写了什么”

预期：
- 墨影直接处理或只调用一个 agent
- 不过度拆解

### 用例 2：复杂任务
示例：
- “调研一下方案，再写汇报，再配图”

预期：
- 墨影先判断为复杂任务
- 再拆给探针 / 笔官 / 片场
- 最终由墨影汇总

### 用例 3：高风险任务
示例：
- “帮我改生产配置并直接发布”

预期：
- 墨影识别为高风险
- 不直接放权
- 先出方案或执行稿，再决定是否放行

- [ ] 高风险复核规则已保留
- [ ] 片场 3 个子 agent 逻辑已明确
- [ ] 至少跑过 1 个简单任务
- [ ] 至少跑过 1 个复杂任务
- [ ] 至少跑过 1 个高风险模拟任务

## 七、建议搭配使用

如果你还需要可视化观察运行状态，可搭配：
- `openclaw-control-center`

这样组合后：
- 本项目负责“团队制度层”
- Control Center 负责“运行观察层”

## 八、结论

最小可用方式：
> 先贴 prompt，先跑起来。

更稳的完整方式：
> prompt + 子 agent 映射 + 片场规则 + 验证流程。
