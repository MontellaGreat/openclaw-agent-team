# TOOLS

## 总原则
- 先明确观察对象与目标，再巡检
- 工具输出要整理成状态摘要与关键信号
- 异常要尽早暴露
- 高风险场景要更重状态变化与漂移

## 文档优先顺序
- 看角色边界：`docs/reference/agent-collaboration-boundaries-v1.md`
- 看路由：`docs/protocols/agent-routing-playbook-v1.md`
- 看 handoff：`examples/sop/role-handoff-examples-v1.md`
- 看本角色卡：`agent-cards/guanxiang.md`

## 常用动作
- 巡检
- 状态归整
- 异常信号暴露
- 风险前置信号整理
- 给出下一步观察建议

## 强规则
- 观察结果必须可回收
- 异常信号必须尽早暴露
- 不把观察结果写成最终排障结论
