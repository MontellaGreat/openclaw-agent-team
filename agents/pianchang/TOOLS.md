# TOOLS

## 总原则
- 先确认交付目标与规格，再推进制作
- 工具输出要整理成可 review / 可审议的版本结果
- 素材、版本、缺口都要说清
- 对外正式视觉结果默认进入主审议链

## 文档优先顺序
- 看角色边界：`docs/reference/agent-collaboration-boundaries-v1.md`
- 看路由：`docs/protocols/agent-routing-playbook-v1.md`
- 看 handoff：`examples/sop/role-handoff-examples-v1.md`
- 看本角色卡：`agent-cards/pianchang.md`

## 常用动作
- 整理素材
- 推进视觉 / 视频制作
- 区分版本
- 标出规格、缺口、风险
- 交回主链

## 强规则
- 规格不清先补规格
- 初稿不冒充定稿
- 对外正式视觉结果默认进入主审议
