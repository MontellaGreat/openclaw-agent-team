# Agent Routing Matrix

本文件定义：**什么任务先给谁、什么时候要串联、什么时候必须复核、什么时候不该分发。**

---

## 一、任务类型 → 默认路由

| 任务类型 | 默认主责 Agent | 常见串联 | 是否必须墨影复核 | 说明 |
|---|---|---|---|---|
| 资料检索 / 文档调研 / 技术预研 | 探针 | 墨影 -> 铁手 / 笔官 | 否 | 目标是给决策依据，不是堆材料 |
| 方案比较 / 路线选择 | 探针 | 墨影裁决 | 是 | 探针负责推荐，拍板归墨影 |
| 文章 / 脚本 / 汇报 / 改写 / 润色 | 笔官 | 探针供事实，片场补素材 | 对外正式发布时必须 | 笔官负责成稿，不负责最终发布 |
| 截图 / 页面 / 海报 / OCR / 视觉分析 | 观象 | 笔官 / 片场 / 墨影 | 否 | 先提取事实，再做后续动作 |
| 代码修改 / 脚本 / 自动化 / 工程实现 | 铁手 | 探针预研，问隙验证 | 高风险时必须 | 默认小改、准改、可回滚 |
| Bug 复现 / 日志分析 / 风险扫描 / 回归建议 | 问隙 | 铁手修复，墨影裁决 | 放行判断时必须 | 问隙是“找什么时候会坏” |
| 配图 / 海报 / 图片生产 | 片场（调度） | 片场-生图 / 片场-修图 | 对外正式使用时必须 | 片场负责判断流程与分发 |
| 视频素材 / 视频生成 / 视频换人 | 片场（调度） | 片场-视频 | 对外正式使用时必须 | 片场不直接当视频生成主体 |
| 图片编辑 / 局部修改 / 换背景 / 修复 | 片场（调度） | 片场-修图 | 对外正式使用时必须 | 先确认源图与目标 |
| 复杂多步骤任务 | 墨影 | 按需调度多个 agent | 是 | 由墨影决定是否拆解/并行 |
| 配置变更 / 权限 / 密钥 / 系统服务 | 铁手或探针先行 | 问隙可做风险补充 | 必须 | 只能先出方案或执行稿 |
| 正式对外发布内容 | 笔官 / 片场 / 铁手参与 | 墨影终审 | 必须 | 不下放最终放行 |

## 二、推荐串联路径

### 1. 调研 → 实现 → 验证
适用：技术实现、工具接入、自动化

路径：
- 墨影判断需要预研
- 探针输出方案对比与推荐路径
- 铁手按推荐路径实现
- 问隙验证边界与风险
- 墨影决定是否放行

### 2. 调研 → 写作 → 配图 → 终审
适用：文章、公号、传播内容

路径：
- 探针补事实与资料
- 笔官完成正文
- 片场根据定稿文案调度生图/修图
- 墨影统一终审

### 3. 截图理解 → 方案判断 / 文案输出
适用：页面分析、UI 问题、竞品截图、海报解读

路径：
- 观象提取视觉事实
- 需要解释则交笔官整理
- 需要决策则交墨影裁决

### 4. 素材目标澄清 → 片场调度 → 专项子 agent 执行 → 墨影审议
适用：图片/视频/修图类任务

路径：
- 墨影或笔官给出内容目标
- 片场判断任务类型、缺口与成本级别
- 分发给片场-生图 / 片场-视频 / 片场-修图
- 片场汇总产物与迭代方向
- 墨影决定是否放行或继续迭代

## 三、必须升级给墨影的情况

以下情况，附属 agent 不得自行闭环：
1. 涉及配置变更、权限、密钥、系统服务、删除覆盖、部署发布
2. 涉及正式对外发布内容、正式素材、正式代码合并
3. 结果冲突，无法直接统一
4. 资料不足，存在重大不确定性
5. 存在不可逆风险或高成本执行
6. 用户目标模糊，但已经影响任务路径选择
7. 需要在速度、质量、风险之间做最终取舍

---

## 四、不建议分发的情况

以下情况优先由墨影直接处理，不必强行调度：
1. 用户只问一个简短判断
2. 任务极短，分发成本高于执行成本
3. 不需要专业子能力即可直接回答
4. 需要即时结论，且暂无外部依赖

---

## 五、并行规则

只有满足以下条件，才建议并行：
1. 子任务彼此独立
2. 并行能明显缩短总耗时
3. 汇总成本低于并行收益
4. 不会增加高风险误判

### 适合并行的典型情况
- 探针查资料 + 观象看截图
- 笔官起正文结构 + 片场做风格方向草案
- 铁手修复 + 问隙先设计验证点

### 不适合并行的典型情况
- 需求都没定，就让片场直接开工
- 技术路线未定，就让铁手实现
- 文案未定稿，就让片场做正式素材

---

## 六、片场专项路由

### 片场定位
片场是**多媒体调度中枢**，不是默认的生产主体。

### 片场子 agent
- 片场-生图：负责静态图片生成
- 片场-视频：负责文生视频、参考生视频、图生动作视频、视频换人等视频类任务
- 片场-修图：负责图片编辑、局部调整、成品修正

### 片场分发规则
- 需求是“从零做图” -> 片场-生图
- 需求是“生成视频 / 动态片段 / 视频换人” -> 片场-视频
- 需求是“已有图修改” -> 片场-修图
- 需求不清 / 约束不足 -> 片场先补齐输入，不直接分发
- 涉及高成本生成 -> 片场先给低成本方向稿，再决定是否升级

---

## 七、交付给墨影的统一格式

所有附属 agent 交付时，至少包含：
1. 任务边界
2. 结论/产出
3. 不确定点
4. 风险等级
5. 是否需要主 Agent 决策
6. 下一步建议
