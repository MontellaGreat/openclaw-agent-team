# Troubleshooting

## 1. 主 Agent 像转发器，不像总控
### 表现
- 只会复述 specialist 结果
- 没有做冲突裁决
- 没有做最终判断

### 解决
- 恢复 `墨影` 的主链：判断 -> 分级 -> 路由 -> 回收 -> 审议 -> 放行
- 明确最终裁决权只在 `墨影`

---

## 2. specialist 越权拍板
### 表现
- `承枢` 直接定方向
- `问隙` 直接宣布正式放行
- `铁手` 自检后直接视为完成

### 解决
- 恢复边界规则
- 所有放行回到主 Agent 审议链

---

## 3. handoff 不完整
### 表现
- 只说“做完了”
- 没有风险说明
- 没有验证方式

### 解决
- 强制使用 `examples/sop/role-handoff-examples-v1.md`
- 至少补齐：产物、验证、风险、未完成项

---

## 4. 没有 review gate 就想 release
### 表现
- 产物还没整理清楚就想直接交付
- 高风险动作没有复核

### 解决
- 先过 `Review Gate`
- 高风险结果必须先复核，再进入主 Agent 审议

---

## 5. 片场乱接任务
### 表现
- 文案没定稿就开始做正式图
- 修图任务被当成生图
- 视频类任务边界不清

### 解决
- 片场先判断任务类型
- 先补齐规格、素材、文案和目标约束
- 不清楚时先出草案，不直接推进正式产物

---

## 6. 高风险任务被直接落地
### 表现
- 改配置直接执行
- 正式对外内容未经审议就发出

### 解决
- 立即恢复高风险规则
- 相关任务只能先出方案 / 草案 / 执行稿
- 必须进入 Review，再由墨影放行

---

## 7. 没有真实子 agent 怎么办？
### 解决
- 先用 prompt-only 模式启动
- 由主 Agent 内部分工模拟
- 等环境允许时，再逐步接入真实子 agent

---

## 8. 项目迁移到别的 OpenClaw 后不顺手
### 原因
- agent 名称不同
- 路径不同
- prompt 注入位置不同

### 解决
- 使用 `INSTALL.md`
- 做角色能力映射，而不是执着于同名
- 先跑一个简单任务 + 一个复杂任务验证

---

## 9. 任务卡住了
### 表现
- 长时间没有结果
- 不知道卡在哪个环节

### 解决
- 回看生命周期：任务当前是 `Assigned`、`In Progress`、`Handoff` 还是 `Review`
- 检查是否缺输入、缺权限、缺依赖
- 必要时将状态标记为 `Blocked`
