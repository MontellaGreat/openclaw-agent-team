# 角色路由执行手册 v1

## 1. 路由原则
- 先判断任务是否清楚
- 再判断是否简单或复杂
- 再判断首选角色
- 最后决定是否需要多角色协作

## 2. 简单任务
### 直接路由
- 纯判断题 -> `墨影`
- 纯成稿题 -> `笔官`
- 纯实现题 -> `铁手`
- 纯测试题 -> `问隙`
- 纯调研题 -> `探针`
- 纯巡检题 -> `观象`
- 纯多媒体题 -> `片场`

## 3. 复杂任务
### 默认链路
- `墨影` 先判断与拆解
- 有多步骤依赖时 -> `承枢`
- specialist 分头执行
- 结果回到 `墨影` 最终裁决

## 4. 高风险任务
### 默认链路
- `墨影` 判断为 `L4`
- `铁手` / `笔官` / `片场` 等负责产出
- `问隙` 负责复核
- `承枢` 视复杂度负责推进和回收
- `墨影` 最终审议是否放行

## 5. 不应误路由的情况
- 不要把 `承枢` 当主 Agent 用
- 不要把 `探针` 当成正式成稿位
- 不要把 `铁手` 当 QA 放行位
- 不要把 `观象` 当最终排障闭环位
- 不要把 `问隙` 的放行建议当最终放行

## 6. 交付前检查
路由结束前至少检查：
- 是否分工过度
- 是否边界重叠
- 是否缺复核角色
- 是否缺最终裁决者
