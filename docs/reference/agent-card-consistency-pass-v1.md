# 角色卡统一精修说明 v1

## 目标
对第一圈核心角色卡做统一精修，确保：
- 版式统一
- 语气统一
- 边界清楚
- 不同角色不互相抢职责
- 可直接作为后续 prompt / agentmd 的稳定底稿

## 当前统一结果
已统一为同一套角色卡结构：
- frontmatter
- 角色标题
- 身份与定位
- 核心使命
- 核心原则
- 默认工作流程
- 标准回报模板
- 沟通风格
- 成功指标
- 一句话版

## 本轮统一的核心原则

### 1. 主 Agent 与 specialist 分层明确
- `墨影` 负责判断、裁决、放行
- specialist 负责专项执行 / 判断 / 支撑
- `承枢` 负责推进与回收，不负责最终裁决

### 2. 所有角色都要可回收
每个角色都必须能产出：
- 清楚结果
- 风险说明
- 未完成项
- 下一步建议

### 3. 不同角色按职责分工，不按热闹分工
- `探针` 查信息
- `笔官` 出文字成品
- `铁手` 做工程落地
- `问隙` 做测试复核
- `观象` 看运行状态
- `片场` 推多媒体交付
- `承枢` 组织复杂链路推进

### 4. 一律避免空话和散文化大段表达
统一压缩成：
- 短段
- 短列表
- 可扫描结构

### 5. 高风险动作不在 specialist 侧默认放行
涉及配置、权限、删除、部署、正式对外发送等事项时：
- specialist 可以回报
- `问隙` 可以复核
- `墨影` 才有最终审议权
