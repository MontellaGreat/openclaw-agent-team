# Review and Quality Gates

本文件定义多 Agent 团队中的复核机制与质量门禁。

---

## 一、目标

质量门禁的作用：
- 防止执行结果未经检查直接放行
- 防止主 Agent 被附属 agent 的结论绑架
- 把“做完”和“可放行”分开

---

## 二、核心原则

1. **产出 ≠ 放行**
2. **交接 ≠ 终审**
3. **高风险任务必须复核**
4. **最好让不是产出者的人做第二双眼检查**
5. **最终放行权永远在墨影**

---

## 三、典型质量门禁

### 1. 探针 → 墨影
- 检查结论是否足以支撑决策
- 检查“确定 / 高概率 / 待验证”是否区分清楚

### 2. 铁手 → 问隙 → 墨影
- 铁手给实现结果
- 问隙做验证与风险评估
- 墨影决定是否放行

### 3. 笔官 → 墨影
- 笔官完成成稿
- 墨影决定是否可对外使用

### 4. 片场 → 墨影
- 片场汇总素材草案或成品
- 墨影决定是否继续迭代或正式放行

---

## 四、必须进入 Review 的情况

以下情况不得跳过 Review：
- 配置变更
- 权限变更
- 密钥处理
- 系统服务修改
- 删除覆盖
- 部署发布
- 正式对外内容
- 正式素材输出
- 用户明确要求“是否建议放行”

---

## 五、Review 输出至少包含

1. 检查对象
2. 检查结论
3. 风险等级
4. 未验证项
5. 是否建议放行

---

## 六、一句话原则

> 能产出不代表能放行；放行前必须过门。
