# Task Complexity Levels

本文件定义任务复杂度分级，用于帮助墨影快速判断：
- 直接执行
- 单 agent 处理
- 多 agent 协作
- 高风险复核

---

## 一、分级体系

### L1 — 直接执行
特点：
- 简单、短、小
- 墨影可稳定直接完成
- 不需要专项 agent
- 风险很低

示例：
- 润色一段话
- 回答一个明确问题
- 整理一小段内容

处理方式：
- 直接执行
- 不拆解

---

### L2 — 单专项处理
特点：
- 需要某个 specialist 的能力
- 但不需要多角色串联
- 风险可控

示例：
- 查某个 API 文档
- 做一次 OCR 提取
- 修一个小 bug

处理方式：
- 派一个附属 agent
- 墨影回收并整合

---

### L3 — 多步骤协作
特点：
- 需要多个能力配合
- 存在调研→执行→验证或写作→配图等链路
- 需要拆解和交接

示例：
- 调研后写汇报
- 修脚本并验证
- 写文章并配图

处理方式：
- 墨影拆解
- 明确 handoff
- 必要时并行
- 汇总后终审

---

### L4 — 高风险协作
特点：
- 涉及高风险边界
- 不可逆、对外、权限、配置、部署等
- 需要明确 review gate

示例：
- 改生产配置
- 改权限
- 正式发布
- 部署上线

处理方式：
- 必须复核
- 附属 agent 只给方案/执行稿/分析
- 最终放行权在墨影

---

## 二、使用原则

1. 先分级，再决定是否拆解
2. 简单任务不要硬升级
3. 高风险任务不要降级处理
4. 若介于两级之间，宁可往高一级保守处理

---

## 三、一句话原则

> 不是所有任务都值得拆；但高风险任务必须严肃对待。
