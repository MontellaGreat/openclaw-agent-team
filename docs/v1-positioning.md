# v1 Positioning

## 这个项目是什么

`openclaw-agent-team` 是一个面向 OpenClaw 的 **多 Agent 协作操作规范包**。

它不是某个具体业务系统，也不是一套绑定单一模型、单一平台、单一组织结构的私有流程。

它的定位是：
- 给主 Agent 提供明确的总控规则
- 给附属 Agent 提供清晰的职责边界
- 给复杂任务提供可复用的拆解、交接、复核、放行框架
- 给多媒体任务提供片场式调度机制
- 给新实例提供 prompt-first 的迁移方式

---

## 它最适合谁

适合：
- 已经在用 OpenClaw，想从“单助手”升级到“有分工的团队协作”
- 已有多个 agent，但职责边界不清
- 希望主 Agent 保留最终审议权
- 希望复杂任务可拆、简单任务可直做
- 希望高风险任务必须复核

---

## 它不是什么

它不是：
- 一个通用自动化平台
- 一个强制所有任务都必须多 agent 的框架
- 一个要求主 Agent 永远不能亲自执行的纯调度系统
- 一个依赖固定 agent 名称才能运行的仓库
- 一个只适合内容生产或只适合代码开发的单场景模板

---

## 它的核心哲学

### 1. 先判断，再调度
主 Agent 先判断任务是否简单、是否复杂、是否高风险。

### 2. 简单任务直接做
不为了显得像团队而拆任务。

### 3. 复杂任务才拆解
拆解必须带来效率、质量或风险控制收益。

### 4. 高风险必须复核
附属 agent 可以分析、实现、验证，但最终放行权不能下放。

### 5. 交接不等于完成
产出之后还需要 handoff、review、裁决、放行。

---

## v1 的边界

v1 重点解决的是：
- 角色定义
- 路由矩阵
- 复杂度分级
- 生命周期
- handoff 协议
- review gate
- 片场调度规则
- prompt-first 迁移

v1 还没有重点覆盖：
- UI 级可视化控制台深度耦合
- 自动化状态同步系统
- 完整 benchmark / 评估框架
- 复杂权限编排系统

---

## 一句话定位

> `openclaw-agent-team` 不是为了把每件事都拆成很多人做，而是为了让 OpenClaw 在该直做时直做、该协作时协作、该复核时复核。
