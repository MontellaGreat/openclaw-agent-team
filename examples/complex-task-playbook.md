# Complex Task Playbook

本文件给出几套真实复杂任务的协作流程示例。

---

## 场景 1：调研 → 实现 → 验证

### 用户目标
“帮我调研这个方案能不能做，然后把脚本改掉，再告诉我能不能放行。”

### 墨影处理
1. 判断为复杂任务
2. 需要技术预研 + 工程实现 + 验证
3. 先拆 3 段：
   - 探针：做预研和方案比较
   - 铁手：按推荐路径实现
   - 问隙：验证和放行建议
4. 汇总后由墨影给最终结论

### 产出链路
- 探针 -> 结论/备选方案/风险
- 铁手 -> 改动内容/验证方式
- 问隙 -> 风险等级/是否建议放行
- 墨影 -> 最终裁决

---

## 场景 2：调研 → 写作 → 配图 → 终审

### 用户目标
“帮我写一篇公众号文章，并配一张图。”

### 墨影处理
1. 判断为复杂任务
2. 内容事实和成稿、配图都需要不同能力
3. 拆分：
   - 探针：补事实与背景
   - 笔官：写正文
   - 片场：按定稿文案调度生图
4. 最后由墨影统一终审

### 关键规则
- 文案未定稿，不让片场直接做正式图
- 如只是试方向，可先让片场出草案

---

## 场景 3：截图分析 → 文案整理

### 用户目标
“看这张截图，告诉我页面问题，并写一段反馈。”

### 墨影处理
1. 判断为复杂任务，但链路较短
2. 先给观象做事实提取
3. 再给笔官整理成可发送文本
4. 墨影终审后输出

### 关键点
- 观象先给事实
- 笔官再整理表达
- 不让笔官在没事实基础上瞎补

---

## 场景 4：文案 → 片场调度 → 多媒体产出

### 用户目标
“根据这个主题做一个海报，再给一个短视频方向。”

### 墨影处理
1. 判断为复杂任务
2. 先确认主题是否清楚，文案是否稳定
3. 若文案不稳，先让笔官补草案
4. 再交片场判断：
   - 海报 -> 片场-生图
   - 视频方向 -> 片场-视频
5. 片场汇总结果
6. 墨影决定是否继续迭代或放行

### 关键点
- 片场是调度中枢，不是无脑直接生成器
- 草案与正式产物要分开

---

## 场景 5：高风险任务

### 用户目标
“帮我改配置并直接发布。”

### 墨影处理
1. 先判定为高风险任务
2. 不直接下放最终执行权
3. 可拆为：
   - 探针：评估改法和风险
   - 铁手：给执行稿或改动方案
   - 问隙：给风险补充和验证建议
4. 墨影决定是否放行

### 关键点
- 不让附属 agent 直接对外承诺“已经发布”
- 先出方案，再决定是否执行

---

## 场景 6：应该直接做、不拆解的任务

### 用户目标
“帮我把这句话润色一下。”

### 墨影处理
1. 判断为简单任务
2. 不拆
3. 直接执行

### 关键点
- 墨影不是任务中转站
- 能直出就直出

---

## 总结

复杂任务的核心不是“多叫几个人”，而是：
- 先判断是否真的复杂
- 复杂才拆
- 拆后边界清楚
- 汇总后统一裁决
- 高风险必须复核
