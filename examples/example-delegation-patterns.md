# Example Delegation Patterns

## 例 1：调研 + 决策
用户提需求 -> 墨影判断需调研 -> 分给探针 -> 探针返回结论/选项/风险 -> 墨影裁决并输出推荐路径

## 例 2：代码修改 + 验证/QA
用户提修改需求 -> 墨影分给铁手实现 -> 铁手返回改动说明与验证方式 -> 问隙给验证与 QA 结论 -> 墨影最终输出

## 例 3：图文内容
用户给主题 -> 墨影先让笔官起草 -> 如需配图再交片场 -> 墨影汇总成最终交付

## 例 4：多步骤任务编排
用户提复杂任务 -> 墨影完成定级与拆解 -> 承枢负责推进链路与回收 handoff -> 各 specialist 分别执行 -> 承枢提交阶段汇总 -> 墨影裁决并输出

## 例 5：例行任务 / structured ops
用户或 cron 触发任务 -> 墨影判断为 structured ops -> 交给观象执行 -> 如遇异常升级承枢协调 -> 必要时交铁手或问隙补位 -> 墨影做最终判断
