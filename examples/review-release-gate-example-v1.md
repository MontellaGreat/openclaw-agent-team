# Review / Release Gate 示例 v1

## 场景
任务已经完成实现，但是否进入正式放行仍需经过 gate。

## Review Gate
### 目标
确认结果是否达到“可被审议”的标准。

### 最低要求
- 已有真实产物
- 已有变更说明
- 已有验证结果或未验证项说明
- 风险点已标出

### 典型参与角色
- `铁手`：提交实现结果
- `问隙`：给复核结论
- `承枢`：在复杂链路中负责回收整理
- `墨影`：决定是否通过 review gate

### 通过条件
- 结果可 review
- 信息足够支撑判断
- 未发现明显阻断问题

### 不通过条件
- 产物不完整
- 复核缺失
- 风险不清
- 关键信息缺口过大

## Release Gate
### 目标
确认结果是否达到“可正式放行”的标准。

### 最低要求
- 已通过 review gate
- 高风险项已复核
- 风险与未完成项已明确
- 主 Agent 完成最终审议

### 典型参与角色
- `问隙`：给放行建议
- `承枢`：给阶段汇总（如为复杂链路）
- `墨影`：做最终放行 / 暂停 / 退回判断

### 通过条件
- 证据足够
- 风险可接受
- 不存在必须先补的阻断项

### 有条件通过
- 主路径通过
- 但仍有明确残留风险或待补项
- 必须把条件写清

### 不通过
- 风险过高
- 关键验证缺失
- 高风险动作仍未经复核
- 主 Agent 判断当前不应放行

