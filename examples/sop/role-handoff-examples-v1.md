# 角色 Handoff 示例集 v1

## 1. 探针 -> 墨影

```text
结论摘要：当前可行方案有 2 条，A 更稳，B 更快但风险更高。
关键依据：已查官方文档、现有仓库结构、历史实现样式。
已确认事实：当前项目已有现成目录结构可复用；目标交付偏向低风险落地。
推断：若优先求稳，应走 A；若优先求速度，可考虑 B。
未知 / 缺口：缺最终验收优先级说明。
下一步建议：请主 Agent 在“稳”与“快”之间拍板。
```

## 2. 墨影 -> 铁手

```text
子任务目标：完成配置文件改造，并保留原结构兼容。
任务边界：只改配置与相关引用，不扩到无关模块。
已给输入：现有配置文件、目标字段、兼容要求。
期望输出：可落地修改 + 变更说明 + 验证方式。
完成标准：配置可读、结构一致、风险项标清。
风险等级：中。
是否需要 review：是。
```

## 3. 铁手 -> 问隙

```text
已完成内容：配置字段已新增，旧字段保留兼容。
产物位置：config/app.json
核心变更：新增主字段、保留 fallback 字段、更新引用读取逻辑。
验证方式：已做静态检查，未做运行态验证。
风险 / 未完成项：运行态尚未实测；高风险变更建议复核。
是否需要主 Agent / 问隙复核：需要。
```

## 4. 问隙 -> 墨影

```text
已验证内容：字段可读取、旧字段未失效、配置格式有效。
未验证内容：服务重启后的运行态表现。
发现的问题：暂无阻断问题。
复现路径：本地读取配置、检查兼容字段行为。
影响范围：配置加载路径。
是否建议放行：有条件放行。
风险等级：中。
```

## 5. 墨影 -> 承枢

```text
任务目标：推动“配置修改 -> 复核 -> 发布判断”三阶段链路。
当前拆解：铁手负责实现，问隙负责复核，最后回主 Agent 裁决。
当前阶段：实现阶段。
关键约束：高风险动作必须复核；最终放行权保留在主 Agent。
期望回收：阶段总结 + 阻塞点 + 是否进入下一阶段建议。
```

## 6. 承枢 -> 墨影

```text
当前阶段：复核阶段已完成。
阶段目标：确认配置变更是否具备进入放行判断条件。
已完成项：铁手完成实现；问隙完成静态复核。
未完成项：运行态重启验证。
已回收 handoff：铁手实现回报、问隙复核回报。
当前阻塞 / 风险：缺服务重启后的最终验证。
是否具备进入下一阶段条件：具备进入主 Agent 审议条件，但不建议直接正式放行。
需要主 Agent 裁决的点：是否先要求运行态验证，再决定最终放行。
下一步建议：先补运行态验证。
```
