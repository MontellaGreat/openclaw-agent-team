# 统一验收样例 v1

## 验收目标
验证一条最小但完整的多角色链路是否成立：
- 主 Agent 判断
- specialist 执行
- QA 复核
- 主 Agent 审议

## 样例任务
**任务类型：** 单文件规则/模板修订 + 关联文档更新

这是当前 v1 阶段最适合作为正式验收的样例，因为：
- 范围清楚
- 风险可控
- 能覆盖判断、执行、复核、放行四个关键动作
- 可以验证 handoff 和路由规则是否真的可跑

## 验收链路

### 1. 墨影
- 判断任务属于 `L2` 或低复杂 `L3`
- 明确目标、边界、验收标准
- 指派 `铁手` 执行
- 必要时指派 `问隙` 复核

### 2. 铁手
- 修改目标文件
- 同步更新关联文档
- 输出变更说明与验证方式

### 3. 问隙
- 检查是否命中目标
- 检查是否漏改关联文档
- 给出放行建议

### 4. 墨影
- 基于证据做最终审议
- 输出“通过 / 有条件通过 / 不通过”

## 通过标准
- 主 Agent 有明确任务判断
- specialist 有真实产物
- QA 有明确复核结果
- 最终放行权没有越权下放
- handoff 信息足够清楚
- 能明确指出风险和未完成项

## 失败标准
- 没有明确任务边界
- 只有分析，没有实际产物
- 没有复核或复核无依据
- specialist 越权宣布正式完成
- 主 Agent 未进行最终审议

## 推荐输出格式

```text
最终结论：
已完成项：
核心依据 / 关键结果：
风险 / 未完成项：
下一步建议：
```
