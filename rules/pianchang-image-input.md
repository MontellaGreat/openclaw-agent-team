# 片场-生图输入规则

来源：用户提供《生成图片使用说明.md》整理。

---

## 一、默认模型策略

片场-生图默认优先级：
1. `flux-merged`：默认推荐，质量较高且速度更快
2. `flux-schnell`：更快，适合草案和快速试图
3. `flux-dev`：适合特定非商业/实验性场景

注意：
- 文档说明 FLUX 相关能力适用于 **中国大陆（北京）地域**
- 必须使用对应地域 API Key
- FLUX 可能存在免费额度、申请开通、调用限时等前置条件

片场调度时必须先判断：
- 当前环境是否已开通模型
- 是否需要草案优先
- 是否需要更快速度还是更高质量

---

## 二、适用任务

适用于：
- 从零生成图片
- 海报 / 封面 / 配图 / 氛围图
- 概念图 / 风格探索图
- 单张图像快速产出

不适用于：
- 已有源图的局部编辑或替换（应转片场-修图）
- 视频或动态素材（应转片场-视频）

---

## 三、最小输入要求

片场在分发给片场-生图前，至少要确认：
1. **prompt**：生成目标描述
2. **size**：分辨率
3. **用途**：海报/封面/配图/草案等
4. **阶段**：草案 / 正式图

建议补齐：
5. 风格关键词
6. 画幅比例
7. 主体/场景/动作/氛围
8. 是否需要固定 seed
9. 是否需要更高创造性或更强遵循

如果以上关键信息缺失，片场不应直接开工。

---

## 四、可控参数

### 必填/核心参数
- `model`
- `prompt`

### 常用可选参数
- `size`：默认 `1024*1024`
- `seed`
- `steps`
- `guidance`：默认 3.5
- `offload`
- `add_sampling_metadata`

### 尺寸约束
支持示例分辨率：
- `512*1024`
- `768*512`
- `768*1024`
- `1024*576`
- `576*1024`
- `1024*1024`

片场应根据用途选尺寸：
- 方图：`1024*1024`
- 竖版：`576*1024` 或 `768*1024`
- 横版：`1024*576`

---

## 五、调度规则

### 规则 1：草案优先
如果用户只是要“看看方向”，优先：
- `flux-schnell`
- 较少 steps
- 默认尺寸

### 规则 2：正式图优先质量
如果是正式产物，优先：
- `flux-merged`
- 明确 size
- 明确 prompt
- 必要时固定 seed 便于复现

### 规则 3：输入不清先补信息
以下情况不得直接生成：
- 只说“帮我做张图”
- 无用途、无风格、无尺寸
- 文案未定但要求直接做正式海报

### 规则 4：结果要提醒时效
文档说明生成结果 URL 可能 **24 小时有效**，片场回传时必须提醒及时保存或转存。

### 规则 5：异步任务要回传状态
若走异步接口，片场-生图回传时需包含：
- `task_id`
- `task_status`
- `results.url`
- `usage.image_count`

---

## 六、片场-生图标准输出

1. 素材目标
2. 所选模型
3. 输入摘要（prompt / size / seed / steps）
4. 产物说明
5. 风格说明
6. 限制与风险
7. 是否建议继续迭代
8. 结果保存提醒
