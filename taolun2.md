# taolun2

## 讨论主题
围绕“智能任务规划系统”专项，进行独立多 Agent 讨论，目标是形成专项计划书，并明确它如何作为项目本体的外部规划能力接入，而不替代主治理链。

## 已定前提
- 项目本体 v1 已完成两轮讨论并收口到正式规划条文
- 智能任务规划系统不替代治理层
- planner 输出必须经过主 Agent 审核后才能进入主链
- 当前阶段优先讨论：它解决什么、边界是什么、怎么接入、怎么验收

## 讨论范围
本专项只讨论：
1. 智能任务规划系统的目标与非目标
2. 输入 / 输出 contract
3. 与主 Agent / 承枢 / runtime 的边界关系
4. 介入时机与否决机制
5. 风险、失败模式、验收方法

## 讨论原则
1. 不重复项目本体已收敛的内容
2. 不把 planner 写成主裁决者
3. 不以“更智能”为理由扩大系统边界
4. 优先讨论可接入、可控、可验收，而不是先讨论炫技能力

## Round 1 主题建议
先回答 5 个问题：
1. 智能任务规划系统到底解决什么痛点
2. 它不应该解决什么，避免越权
3. 它的最小输入和最小输出是什么
4. 它什么时候可以介入，什么时候必须退出
5. 它的结果如何被主 Agent 审核并接入 runtime
