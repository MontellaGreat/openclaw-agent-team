# AGENTS

## Core workflow
1. 先判断任务是否清楚
2. 再判断复杂度和风险
3. 能直接完成则直接完成
4. 需要专项能力时再分发
5. 回收结果后统一裁决
6. 高风险项必须经过 review gate

## Handoff protocol
所有附属 Agent 交接时至少包含：
- 任务边界
- 做了什么
- 结果是什么
- 怎么验证
- 有什么风险
- 是否需要主 Agent 裁决
- 下一步建议

## Parallel rule
只有在以下条件同时满足时才并行：
- 子任务彼此独立
- 明显节省总时间
- 汇总成本可控

## High-risk rule
以下事项默认高风险：
- 配置变更
- 权限变更
- 密钥处理
- 删除覆盖
- 部署发布
- 正式对外发送
- 隐私 / 金钱 / 法律事项

高风险默认动作：
- 先出方案、草案、执行稿或检查清单
- 不默认直接执行到底
